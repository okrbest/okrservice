클라이언트 포털 위젯 테스트

이 파일은 브라우저에서 파일로 직접 열어도 되지만, 그때는 스크립트 로드가 막히는 경우가 많습니다. 권장: http://127.0.0.1:3200/static/clientportal-widget-test.html (위젯 서버가 3200일 때). file:// 로 연 경우 자동으로 이 주소로 이동합니다(포트가 다르면 파일을 ?widgetHttpOrigin=http://127.0.0.1:내포트 와 함께 열거나, 직접 HTTP 주소를 쓰세요). 배포 URL을 쓰려면 ?root=http://localhost:3200 처럼 지정하세요 (5240help.html 처럼 스크립트만 절대 URL이면 file 열기 가능).

번들을 받으려면 포트 3200에서 위젯 개발 서버가 떠 있어야 합니다. 보통 ./bin/okrservice.js dev … 로 전체 띄울 때 configs.json에 widgets가 켜져 있으면 pm2가 widgets에서 yarn dev를 같이 실행합니다(그걸 수동으로 또 칠 필요는 없습니다). widgets를 설정에서 끈 경우에만 cd widgets && yarn dev가 필요합니다. HTTP로 열 때는 http://localhost:3200/static/clientportal-widget-test.html 도 같습니다.

적용 후 화면 오른쪽 아래의 파란 포털 버튼을 눌러 패널을 엽니다. 개발자 도구 Network는 상단 컨텍스트를 iframe(고객 포털)로 바꿔야 /graphql 요청이 보입니다. 「포털」클릭만으로는 부모 페이지 네트워크에 새 줄이 없을 수 있습니다 (postMessage만 쓰기 때문).
