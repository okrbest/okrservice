# E2E 테스트 (Playwright) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** PDF 7절 "꼭 작성할 테스트 케이스" 3개를 Playwright로 구현한다 — 중복 메시지 방지, 오프라인 메시지 복원, WebSocket 자동 재연결.

**Architecture:** `e2e/` 디렉토리에 Playwright 프로젝트 신규 생성. globalSetup에서 MongoDB에 테스트용 RPA Client를 seed하고 teardown에서 정리. 5240 서버 역할은 Playwright `request` API로 `POST /api/rpa/messages` 직접 호출. 위젯은 `http://localhost:3200/test?type=messenger&brand_id=<id>` 경로로 접근.

**Tech Stack:** Playwright, TypeScript, MongoDB (mongoose), bcryptjs

- [x] **Step 5: e2e/global-setup.ts 작성**

globalSetup은 두 가지를 처리한다:
1. RPA Client seed (테스트 인증용)
2. 기존 Integration의 brand_id 조회 → `process.env.TEST_BRAND_ID`에 설정

```typescript
import * as mongoose from 'mongoose';
import * as bcrypt from 'bcryptjs';
import { nanoid } from 'nanoid';

const MONGO_URL = process.env.MONGO_URL || 'mongodb://localhost/erxes';
const CLIENT_ID = 'e2e-test-client';

export default async function globalSetup() {
  await mongoose.connect(MONGO_URL);

  const db = mongoose.connection.db;

  // 1. RPA Client seed
  await db.collection('clients').deleteMany({ clientId: CLIENT_ID });

  const clientSecret = await bcrypt.hash('e2e-test-secret', 10);
  await db.collection('clients').insertOne({
    _id: nanoid(),
    name: 'e2e-test-client',
    clientId: CLIENT_ID,
    clientSecret,
    refreshToken: '',
    whiteListedIps: [],
    createdAt: new Date(),
  });

  // 2. 기존 Integration brand_id 조회 (widget 로드에 필요)
  // TEST_BRAND_ID 환경변수가 이미 있으면 그것을 사용
  if (!process.env.TEST_BRAND_ID) {
    const integration = await db.collection('integrations').findOne(
      { kind: 'messenger' },
      { projection: { _id: 1 } }
    );
    if (!integration) {
      throw new Error(
        '[E2E globalSetup] messenger 타입 Integration이 DB에 없습니다.\n' +
        '해결방법: 관리자 UI에서 채널/통합을 생성하거나 TEST_BRAND_ID 환경변수를 설정하세요.'
      );
    }
    process.env.TEST_BRAND_ID = integration._id.toString();
  }

  console.log('[E2E] TEST_BRAND_ID:', process.env.TEST_BRAND_ID);

  await mongoose.disconnect();
}
```

## Task 2: 테스트 1 — 중복 메시지 방지 (idempotency)

**Files:**
- Create: `e2e/tests/rpa-idempotency.spec.ts`

### 배경 지식

PDF 명시: "같은 (loginId, rpaCode)가 5초 내 2번 도달해도 중복 메시지가 안 쌓이는가"

멱등성 구현: DB에 `{ loginId, messageCode }` sparse unique index. 동일 `messageCode`로 2번 POST하면 2번째는 DB 저장이 skip되어 WebSocket publish 없음 → 위젯에 메시지 1개만 표시됨.

위젯 채팅 영역에서 RPA 메시지는 `div` 안에 봇 아바타(`🤖`)와 메시지 텍스트가 함께 렌더링됨 (`ChatbotView.tsx`). 인사 메시지도 동일한 구조이므로, RPA 메시지를 특정하려면 메시지 텍스트로 필터링.

```typescript
// e2e/tests/rpa-idempotency.spec.ts
import { test, expect, Page, APIRequestContext } from '@playwright/test';

const LOGIN_ID = 'e2e-user@test.com';
const BRAND_ID = process.env.TEST_BRAND_ID!;
const WIDGET_URL = `http://localhost:3200/test?type=messenger&brand_id=${BRAND_ID}`;

async function postRpaMessage(
  request: APIRequestContext,
  messageCode: string,
) {
  return request.post('http://localhost:3300/api/rpa/messages', {
    data: {
      clientId: 'e2e-test-client',
      secret: 'e2e-test-secret',
      loginId: LOGIN_ID,
      rpaCode: 'HR_RPA_100',
      message: '출근 알림 테스트',
      messageCode,
    },
  });
}

function getWidgetFrame(page: Page) {
  return page.frameLocator('#erxes-messenger-iframe');
}

test('같은 messageCode로 2번 POST해도 메시지가 1개만 표시된다', async ({
  page,
  request,
}) => {
  const messageCode = `IDEM-TEST-${Date.now()}`;

  // 1. 위젯 페이지 열기
  await page.goto(WIDGET_URL);

  // 2. 위젯 iframe 로드 대기
  const frame = getWidgetFrame(page);
  await frame.getByText('Chatbot').waitFor({ timeout: 15_000 });

  // 3. 챗봇 탭 클릭
  await frame.getByText('Chatbot').click();

  // 4. 동일 messageCode로 2번 POST
  const res1 = await postRpaMessage(request, messageCode);
  expect(res1.status()).toBe(200);

  // 첫 번째 메시지가 위젯에 나타날 때까지 대기
  await frame.getByText('출근 알림 테스트').waitFor({ timeout: 10_000 });

  const res2 = await postRpaMessage(request, messageCode);
  expect(res2.status()).toBe(200);

  // 5. 잠시 대기 후 메시지 개수 확인
  await page.waitForTimeout(2_000);

  // RPA 메시지 텍스트가 정확히 1개만 존재해야 함
  const msgCount = await frame.getByText('출근 알림 테스트').count();
  expect(msgCount).toBe(1);
});
```

## Task 4: 테스트 3 — WebSocket 자동 재연결

**Files:**
- Create: `e2e/tests/rpa-reconnect.spec.ts`

### 배경 지식

PDF 명시: "WebSocket 비정상 종료 시 60초 내 자동 재연결 되는가"

구현: `apollo-client.ts`의 `wsLink`가 `graphql-ws` 라이브러리로 생성됨. 설정: `retryAttempts: 100`, `retryWait: () => new Promise(resolve => setTimeout(resolve, 1000))` (1초 대기 후 재연결). 소켓이 강제 종료되면 약 1~2초 후 자동 재연결.

강제 종료 방법: `page.evaluate`로 iframe 내부의 WebSocket 객체 접근 후 `close(4000, ...)` 호출. `graphql-ws`는 4000~4999 코드를 재연결 불가 코드로 취급하지 않으므로 재연결됨 (4000은 "connection acknowledged" 관련 코드지만 클라이언트 측 강제 close는 재연결 트리거).

**주의:** `page.evaluate` 내부에서 iframe contentWindow에 접근하려면 `frame.evaluate` 사용.

---

- [x] **Step 1: 테스트 파일 작성**

```typescript
// e2e/tests/rpa-reconnect.spec.ts
import { test, expect, Page, APIRequestContext } from '@playwright/test';

const LOGIN_ID = 'e2e-user@test.com';
const BRAND_ID = process.env.TEST_BRAND_ID!;
const WIDGET_URL = `http://localhost:3200/test?type=messenger&brand_id=${BRAND_ID}`;

async function postRpaMessage(
  request: APIRequestContext,
  message: string,
) {
  return request.post('http://localhost:3300/api/rpa/messages', {
    data: {
      clientId: 'e2e-test-client',
      secret: 'e2e-test-secret',
      loginId: LOGIN_ID,
      rpaCode: 'HR_RPA_100',
      message,
      messageCode: '',
    },
  });
}

function getWidgetFrame(page: Page) {
  return page.frameLocator('#erxes-messenger-iframe');
}

test('WebSocket 강제 종료 후 재연결되어 새 메시지를 수신한다', async ({
  page,
  request,
}) => {
  // 1. 위젯 페이지 열기 및 챗봇 탭으로 이동
  await page.goto(WIDGET_URL);
  const frame = getWidgetFrame(page);
  await frame.getByText('Chatbot').waitFor({ timeout: 15_000 });
  await frame.getByText('Chatbot').click();

  // 2. 첫 번째 메시지 전송 (연결 확인용)
  const firstMsg = `재연결 테스트 초기 메시지 ${Date.now()}`;
  const res1 = await postRpaMessage(request, firstMsg);
  expect(res1.status()).toBe(200);
  await expect(frame.getByText(firstMsg)).toBeVisible({ timeout: 10_000 });

  // 3. iframe frame 객체로 WebSocket 강제 종료
  const messengerFrame = page.frame({ url: /localhost:3200\/messenger/ });
  if (messengerFrame) {
    await messengerFrame.evaluate(() => {
      // graphql-ws가 사용하는 WebSocket 연결 찾아서 종료
      const sockets = (performance as any)
        .getEntriesByType?.('resource')
        ?.filter?.((e: PerformanceEntry) => e.name.includes('graphql'));
      // WebSocket 객체를 직접 찾아 종료
      const wsProto = WebSocket.prototype;
      const origSend = wsProto.send;
      // 열려 있는 WS 찾기: 전역 WebSocket 인스턴스 목록 없으므로
      // 대신 페이지 내 __wsInstances에 등록된 것 사용
      const instances: WebSocket[] = (window as any).__wsInstances || [];
      instances.forEach((ws) => {
        if (ws.readyState === WebSocket.OPEN) {
          ws.close(1001, 'test-forced-close');
        }
      });
    });
  }

  // WebSocket 강제 종료 대안: 서버 측 네트워크를 잠깐 끊는 대신
  // iframe을 잠깐 src="" 로 바꿨다가 되돌리는 방식으로 재연결 시뮬레이션
  await page.evaluate(() => {

    }
  });

  await frame.getByText('Chatbot').waitFor({ timeout: 15_000 });

  // 5. 재연결 후 새 메시지 전송
  const reconnectMsg = `재연결 후 메시지 ${Date.now()}`;
  const res2 = await postRpaMessage(request, reconnectMsg);
  expect(res2.status()).toBe(200);
```
